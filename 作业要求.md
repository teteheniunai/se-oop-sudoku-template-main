# Sudoku/Game 领域对象重构与 Undo/Redo

## 一、作业背景

本次作业以给定的 Sudoku 前端工程为基础，完成一次“面向对象的领域重构”，并实现 **Undo/Redo** 功能。

本次作业的重点不是 UI 美化，也不是增加大量新功能，而是通过一个小而完整的交互系统，理解并实践以下面向对象编程要点：

- 对象职责划分
- 对象生命周期
- 深拷贝 / 浅拷贝
- 串行化 / 反串行化
- 外表化（如 `toString()`、`toJSON()`）
- 历史记录与撤销 / 重做

---

## 二、作业目标

请基于现有工程，完成以下目标：

1. 将核心逻辑重构为清晰的领域对象设计。
2. 至少抽取出以下两个核心对象：
   - `Sudoku`（或 `Board`）
   - `Game`
3. 实现用户输入后的 **Undo / Redo**。
4. 支持当前局面的 **序列化 / 反序列化**。
5. 为核心对象提供合理的 **外表化表示**。
6. 在文档中说明你的对象设计。

## 三、允许的实现方式

你可以使用以下任意一种风格实现：

- **ES6 class**
- **Function-based factory**

例如：

- `class Sudoku { ... }`
- `function createSudoku(...) { ... }`

评分不偏向某一种语法形式。  
评分重点在于：

- 对象边界是否清晰
- 职责划分是否合理
- 行为是否正确
- 设计说明是否自洽

## 四、建议的对象模型

### 1. `Sudoku` / `Board`

表示当前数独局面，是本作业的核心领域对象。

建议其承担如下职责：

- 持有 grid 数据
- 对外提供 `guess(...)` 或等价操作
- 提供局面校验能力
- 提供串行化 / 反串行化能力
- 提供外表化表示

### 2. `Game`

表示“一局游戏”或“一个游戏会话”。

建议其承担如下职责：

- 持有当前 `Sudoku`
- 管理历史记录
- 提供 `undo()` / `redo()`
- 管理保存 / 恢复
- 作为 UI 层与领域对象层之间的主要接口

### 3. `Move` / `Guess`

建议作为简单值对象使用，例如：

```ts
type Move = {
  row: number
  col: number
  value: number | null
}
````

你也可以根据需要扩展字段，但不建议将其设计成过重的核心对象。

## 五、功能要求

### A. OO 重构

你需要将项目中的核心逻辑整理为清晰的对象结构，而不是将主要逻辑继续散落在组件代码中。

最低要求：

* 能明确看到 `Sudoku` / `Game` 的实现
* 关键行为不是直接写在 UI 组件中
* UI 层主要负责渲染与事件转发

---

### B. Undo / Redo

你必须实现：

1. **Undo**

    * 撤销最近一次用户输入

2. **Redo**

    * 在执行 Undo 后，可以恢复被撤销的操作

3. 正确的历史行为

    * 新开一局后，历史清空
    * Undo 后若进行新的输入，Redo 历史应失效
    * 连续 Undo / Redo 行为正确

---

### C. 串行化 / 反串行化

你必须支持：

* 将当前局面导出为 JSON
* 从 JSON 恢复出局面

最低要求：

* `Sudoku` 或 `Game` 提供 `toJSON()` 或等价接口
* 提供 `fromJSON(...)`、`load(...)` 或等价恢复方式

要求你在设计文档中说明：

* 哪些字段会被序列化
* 哪些字段不会被序列化
* 恢复时如何重建对象

---

### D. 外表化

你必须为核心对象提供至少一种清晰的文本表示方式。

建议：

* `Sudoku.toString()`
* `Sudoku.toJSON()`
* `Game.toString()`（可选）

要求：

* 输出结果可用于调试
* 能清晰表达当前局面信息
* 不允许只是简单输出 `[object Object]`

## 九、统一评分接口

学生必须在 `src/domain/index.js` 中导出以下函数：

```js
export function createSudoku(input)
export function createSudokuFromJSON(json)
export function createGame({ sudoku })
export function createGameFromJSON(json)
```

### `createSudoku(input)`

- 输入：`number[][]` 的 9x9 grid
- 输出：一个 Sudoku 对象

Sudoku 对象必须至少提供：

```js
{
  getGrid(): number[][]
  guess(move): void
  clone(): SudokuLike
  toJSON(): any
  toString(): string
}
```

其中：

```js
move = { row: number, col: number, value: number }
```

### `createGame({ sudoku })`

- 输入：一个 Sudoku 对象
- 输出：一个 Game 对象

Game 对象必须至少提供：

```js
{
  getSudoku(): SudokuLike
  guess(move): void
  undo(): void
  redo(): void
  canUndo(): boolean
  canRedo(): boolean
  toJSON(): any
}
```

`createSudokuFromJSON(json)` 与 `createGameFromJSON(json)` 用于反序列化恢复。
